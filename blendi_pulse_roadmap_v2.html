📝 What changed from v1 → v2

Every decision below was confirmed by Jon directly or approved through the Gemini–Claude strategy session. No changes were made without executive input.

Version 2 — Change log
RemovedPulse Level (habit bar) — Removed by Gabriel. Required manual blend registration — if user forgets, streak breaks unfairly. Replaced by Goal Rings (positive progress, not anxiety).
RemovedCommunity Pulses feed — App is a performance tool, not a social network. Replaced by Share Cards only (UGC for external platforms). Tab renamed from "Pulses" to "Track".
RemovedHardware Health Estimation — No direct sensors on the hardware. Any motor health estimate would be inaccurate and could cause bad UX. Cleaning reminder by time (every 7 days) stays — by usage count is removed.
AddedNutrition Goal Tracking — macro rings — Jon's direct request. User sets daily protein/calorie targets in onboarding. Home screen shows Goal Rings filling as blends are logged. Replaces Pulse Level as the main daily engagement driver.
AddedHydration & Supplement Stack tracking — Jon's direct request. 1-tap water log + "My Stack" daily checklist (creatine ✓, collagen ✓). Zero AI cost. High daily-open value for Gym & Protein and Active Moms segments.
MovedOffline Mode → Phase 1 — Jon's direct request. Was not in the original roadmap at all. Cache favorites + last 20 recipes locally. Critical for kitchen use and travel. Jon said "this is Phase 1, not Phase 5."
Addedi18n from Day 0 — Gemini's technical recommendation (confirmed). No hardcoded strings anywhere. Use i18next + JSON language files. Adding Portuguese or Spanish later must be a config change, not a code refactor.
ChangedTab "Pulses" → "Track" — Navigation stays at 5 tabs. The Track tab hosts: Goal Rings detail, Hydration Log, Supplement Stack, blend history. App pivots from recipe tool to daily wellness tracker.
ChangedPantry Scanner — scope reduced for Phase 2 — Focus on common household ingredients first (fruits, oats, vegetables). Supplement Detector (protein tubs, creatine) moves to Phase 5. Smart Substitutes Engine is the main value prop.
AddedSkeleton screens as implementation standard — Every screen with an AI call (Pulse AI, Pantry Scanner) must show skeleton placeholders while loading. App must feel as fast as the 120W hardware — no spinners, no blank screens.

🏗 App architecture

Three independent layers communicating via API. This separation allows the mobile app to evolve independently from the backend — and vice versa. The i18n layer now runs across all three.

Layer 1 — Mobile frontend (what the user sees)

React Native + Expo. Consumes backend API over HTTPS. All UI/UX lives here. HealthKit / Health Connect integration at this layer. i18next handles language switching client-side. Offline cache via React Query + MMKV storage.

Layer 2 — Backend API (the brain)

Node.js + Express. Authentication, database, business logic, AI API calls, payment processing, email delivery, cron jobs (daily push, weekly report). Never exposed directly to the end user.

Layer 3 — Database + Storage (memory)

MongoDB Atlas for structured data (users, recipes, blend logs, macro goals, hydration, supplement stack). Cloudflare R2 for files (Pantry Scanner photos, generated share cards). Upstash Redis for AI response caching and rate limiting.

⚠The current prototype is plain HTML with localStorage. It will NOT be the base of the mobile app. It serves as a UI reference. Development starts from scratch in React Native. The prototype can be deployed to Vercel as a stakeholder demo.

🧠 AI integrations

GPT-4o — Pulse AI + Smart Substitutes + Biometric context

Input: user's BLENDi model, goal, blend history, available ingredients, and biometric data category (if synced). Output: structured JSON with recipe title, ingredients, macros (protein/carbs/fat/kcal), blend time, instructions. System prompt always forces JSON output format.

GPT-4o Vision — Pantry Scanner (Phase 2 scope)

Input: base64 photo. Output: JSON list of detected common ingredients (fruits, vegetables, oats, dairy). Second call generates 3 recipes with Smart Substitutes for missing items. Supplement Detector (protein tubs, creatine) intentionally excluded — Phase 5 only.

Biometric logic — no separate model needed

Backend reads sleep hours + HRV + calories from HealthKit/Health Connect. Maps to simple categories (recovery needed / high energy / dehydration risk). These categories are injected as context into GPT-4o. No dedicated ML model — GPT-4o handles interpretation.

💡Estimated AI cost at MVP (100 active users, 5 Pulse AI requests/week, 10 Pantry scans/month): ~$15–40/month. Redis cache prevents duplicate API calls. Scales linearly with users.
